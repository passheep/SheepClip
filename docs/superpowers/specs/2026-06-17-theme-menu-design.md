# SheepClip 左侧主题菜单设计

## 背景

SheepClip 现有主界面已经有左侧导航、剪贴板、快捷输入、设置等视图。用户希望在当前基础上新增左侧“主题”菜单，支持一键切换几套内置主题颜色，并支持选择本机已安装的内置候选字体。

当前项目使用 Vue 3 + TypeScript + Tauri 2，用户设置通过 `AppSettings` 写入 SQLite。主题功能应沿用现有设置保存链路，不引入新的状态管理库。

## 目标

- 左侧导航新增“主题”入口，和剪贴板、快捷输入、设置同级。
- 颜色主题使用内置主题，不提供自定义取色器。
- 每个主题用卡片展示，卡片里用简单 `div` 和 CSS 描绘大体界面颜色，包括侧栏、内容区、按钮或选中态。
- 字体使用内置候选项，候选字体包括系统默认、微软雅黑、黑体、宋体、楷体等。
- 启动或进入主题页时检测本机是否可用对应字体，不可用字体不展示，也不允许保存。
- 已保存字体如果当前系统不可用，自动回退到系统默认。
- 主题和字体选择需要持久化，重启应用后仍然生效。

## 非目标

- 不做用户自定义颜色输入。
- 不下载或内置字体文件。
- 不改变剪贴板、快捷输入、托盘、快捷键等业务逻辑。
- 不重构整体 UI 结构，只把颜色和字体逐步迁移到主题变量。

## 方案选择

推荐方案是新增独立主题页，并把主题和字体作为 `AppSettings` 的字段保存。

备选方案一是把主题选项放在现有设置页中。它改动小，但左侧主题入口不明显，也不符合本次“左侧增加主题菜单”的目标。

备选方案二是做自定义取色器。它自由度高，但需要更多校验和对比度兜底，容易让界面出现不可控配色。本期先做内置主题更稳。

## 页面设计

左侧导航新增“主题”按钮，使用现有 lucide 图标风格，未展开侧栏时只显示图标，展开时显示文字。

主题页分为两个区域：

1. 颜色主题
   - 使用网格卡片展示内置主题。
   - 卡片上半部分是小型界面预览，用多个 `div` 表示侧栏、顶部栏、列表项、按钮、选中态。
   - 卡片下半部分显示主题名称和简短说明。
   - 当前主题卡片显示选中边框和勾选图标。
   - 点击卡片立即应用并保存。

2. 字体
   - 使用按钮组或列表卡片展示当前系统可用字体。
   - “系统默认”始终展示。
   - 微软雅黑、黑体、宋体、楷体等字体只有检测通过才展示。
   - 点击字体项立即应用并保存。
   - 如果保存中的字体不可用，界面显示系统默认。

## 内置主题

第一期提供 5 套主题：

- 默认暖灰：保留当前 SheepClip 的暖灰、墨色、薄荷和锈红气质。
- 清爽蓝：偏办公工具感，主色为蓝，背景保持浅色。
- 薄荷绿：更轻快，强调色偏绿色，适合长时间使用。
- 石墨灰：更克制，侧栏和强调色偏深灰，适合低干扰。
- 柔和紫：少量紫色作为强调，整体仍保持浅色和可读性。

主题字段保存为稳定 key，例如 `warm`, `blue`, `mint`, `graphite`, `violet`。

## 字体检测

前端提供 `isFontAvailable(fontFamily: string)` 工具函数。检测方式：

- 使用一段包含中文和英文的样本文字。
- 分别用候选字体和几个通用 fallback 字体渲染到隐藏 canvas 或测量元素。
- 如果候选字体渲染宽度与 fallback 明显不同，则认为可用。
- 对“系统默认”不做检测，始终可选。

保存前再次校验当前字体是否在可用列表中。如果不可用，写入系统默认。

## 数据流

`AppSettings` 增加：

- `theme_key: string`
- `font_key: string`

Rust 侧：

- `recommended_settings` 默认 `theme_key = "warm"`，`font_key = "system"`。
- `read_settings` 读取旧数据库时提供默认值。
- `sanitize_settings` 限制主题和字体 key 只能落在内置允许列表内。
- `settings_entries` 写入新增字段。

前端：

- `types.ts` 增加字段类型。
- `commands.ts` 的 fallback settings 增加默认值。
- `App.vue` 读取设置后根据 `theme_key` 和 `font_key` 生成 CSS 变量。
- 主题页点击卡片后更新 `settings`，沿用现有 debounce 保存。

## 样式策略

把当前硬编码的主色和背景逐步迁移到 CSS 变量：

- `--app-font-family`
- `--color-ink`
- `--color-panel`
- `--color-sidebar`
- `--color-header`
- `--color-line`
- `--color-primary`
- `--color-primary-soft`
- `--color-danger`
- `--color-card`
- `--color-muted`

Tailwind 仍用于布局和尺寸，主题相关颜色优先使用 CSS 变量类或行内 style 绑定，避免为每套主题生成大量 class。

## 错误处理

- 如果设置里保存了未知主题 key，自动回退默认暖灰。
- 如果设置里保存了不可用字体 key，自动回退系统默认。
- 如果字体检测失败，保留系统默认和已确认可用字体，不阻塞页面加载。
- 如果保存设置失败，沿用现有设置保存失败逻辑，恢复后端返回的设置。

## 测试和验证

优先增加前端单元测试，覆盖：

- 主题 key 解析和默认回退。
- 字体候选过滤。
- 不可用字体保存前回退系统默认。

实现后运行：

- `npm --cache C:\000Program\SheepClip\.npm-cache run build`
- 如触及 Rust 设置结构，补跑 `cargo check`

如启动本地页面验证，检查：

- 左侧新增主题菜单可进入。
- 主题卡片预览颜色与应用后的界面主色一致。
- 切换主题后侧栏、顶部栏、选中态、按钮、焦点色跟随变化。
- 字体列表只展示系统可用项。
- 重启或刷新后主题和字体仍保持。
